# План реализации проекта TechManualRAG

## 1. Идея и описание
**TechManualRAG** – система мультимодальной обработки технических руководств.
Компактная RAG-система, специализирующаяся на поиске в инженерной документации. Ориентирована на извлечение знаний из PDF-инструкций, где текст сопровождается схемами, таблицами и математическими формулами.

## 2. Уникальные особенности
- **Мультимодальное извлечение данных:** использование OCR и специализированных моделей для распознавания таблиц, формул и связывания текста с диаграммами.
- **Комплексная база знаний:** возможность ссылаться не только на текстовый блок, но и на фрагменты визуальных схем (показ номеров на схеме).
- **Визуальный интерфейс:** пользователь видит найденные изображения и таблицы вместе с текстовым ответом в UI.
- **Легковесность:** ориентация на локальное развертывание без избыточной инфраструктуры (использование квантованных GGUF-моделей с учетом ограничений VRAM).

## 3. Выбор технологий
- **Фреймворк RAG:** LlamaIndex (с мультимодальным функционалом).
- **Модели (Генерация):** Локальная LLM с поддержкой vision (например, LLaVA или Llama-3-Vision через Ollama).
- **Обработка данных:** pytesseract, легковесные альтернативы Nougat (например, Marker) для формул, библиотеки для парсинга PDF (PyMuPDF).
- **Векторная БД:** Qdrant (раздельные эмбеддинги: BGE для текста, CLIP для изображений).
- **Интерфейс:** Streamlit.

## 4. Поэтапный план разработки

### Этап 1. Обработка PDF и мультимодальных данных
- Написать парсеры для извлечения текста, кропа изображений и таблиц из PDF (добавить fallback-стратегию растеризации через `pdf2image` для векторных схем).
- Настроить OCR для текста на схемах.
- Реализовать извлечение и распознавание математических формул.

### Этап 2. Векторизация и индексация
- Сгенерировать эмбеддинги для текста и изображений.
- Связать текстовые чанки с соответствующими изображениями (метаданные) на основе упоминаний в тексте (например, "Рис. 1"), близости bounding box и номеров страниц.

### Этап 3. Интерфейс поиска
- Разработать логику мультимодального поиска (гибридный поиск: BM25 для текста + dense retrieval для текста и изображений).
- Создать UI, который по запросу выдает и сгенерированный текст, и релевантные скриншоты схем.

### Этап 4. Тестирование
- Написать unit-тесты для компонентов парсинга PDF из Этапа 1.
- Подготовить тестовые наборы (руководства пользователя, схемы приборов).
- Оценить точность находимых схем в ответ на сложные технические запросы с использованием автоматизированных метрик (например, Recall@K, MRR).

### Этап 5. Документация и публикация
- Подготовить `README.md` с описанием ограничений и инструкцией по добавлению новых руководств.
- Выложить код на GitHub.
